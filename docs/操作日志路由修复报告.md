## 📋 问题描述

操作日志 API 返回 NOT_FOUND（404），导致 4 个测试用例失败：
- API-LOGS-001: 正常场景 - 默认分页
- API-LOGS-002: 正常场景 - 类型筛选
- API-LOGS-004: 异常场景 - 无 Token
- API-LOGS-005: 正常场景 - 获取日志类型

**根本原因**: 后端路由注册路径与测试用例期望路径不一致

---

## 🔍 问题分析

### 测试用例期望的路径
```
GET /api/admin/audit-logs
GET /api/admin/audit-logs/types
GET /api/admin/audit-logs/stats
GET /api/admin/audit-logs/:id
```

### 修复前的后端路由注册
**文件**: `/root/.openclaw/workspace/projects/clearspring-v2/api/routes/admin/index.js`

```javascript
// ❌ 错误：注册为 /logs
router.use('/logs', auditLogsRoutes);
```

实际访问路径：`/api/admin/logs`（与测试用例不匹配）

---

## ✅ 修复方案

### 修改内容

**文件**: `api/routes/admin/index.js`

**修改前**:
```javascript
// 审计日志路由
try {
    const auditLogsRoutes = require('./audit-logs');
    router.use('/logs', auditLogsRoutes);  // ❌ 路径错误
    logger.info('Admin AuditLogs route loaded');
} catch (err) {
    logger.error('Admin AuditLogs route load failed', { error: err.message });
}
```

**修改后**:
```javascript
// 审计日志路由
try {
    const auditLogsRoutes = require('./audit-logs');
    router.use('/audit-logs', auditLogsRoutes);  // ✅ 路径正确
    logger.info('Admin AuditLogs route loaded');
} catch (err) {
    logger.error('Admin AuditLogs route load failed', { error: err.message });
}
```

**同时更新了根路由返回的端点信息**:
```javascript
// 修改前
logs: 'GET /api/admin/logs'

// 修改后
logs: 'GET /api/admin/audit-logs'
```

## 📝 路由配置文档

### 操作日志模块完整路由表

| 方法 | 路径 | 说明 | 认证要求 |
|------|------|------|----------|
| GET | `/api/admin/audit-logs` | 操作日志列表（支持分页、筛选） | 管理员 |
| GET | `/api/admin/audit-logs/types` | 获取日志类型列表 | 管理员 |
| GET | `/api/admin/audit-logs/stats` | 操作日志统计 | 管理员 |
| GET | `/api/admin/audit-logs/:id` | 日志详情 | 管理员 |
| DELETE | `/api/admin/audit-logs` | 清空日志（需要超级管理员权限） | 超级管理员 |

### 查询参数

**GET /api/admin/audit-logs**
- `page` (number): 页码，默认 1
- `pageSize` (number): 每页数量，默认 20
- `type` (string): 日志类型筛选
- `userId` (string): 用户 ID 筛选
- `startDate` (string): 开始日期 (ISO 8601)
- `endDate` (string): 结束日期 (ISO 8601)
- `keyword` (string): 关键词搜索（用户名）

**GET /api/admin/audit-logs/stats**
- `startDate` (string): 开始日期
- `endDate` (string): 结束日期
- `groupBy` (string): 分组方式 (day/week/month)，默认 day
