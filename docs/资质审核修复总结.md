## 📋 问题描述

资质审核 API 的 7 个测试用例全部失败，返回 `INTERNAL_ERROR`。

**根本原因**: 代码中使用的是 `certificates` 集合，但实际应该使用 `qualifications` 集合。

---

## 🔧 修复内容

### 修改的文件

1. **api/routes/admin/qualifications.js** (核心接口)
   - GET /api/admin/qualifications - 资质列表查询
   - PUT /api/admin/qualifications/:id - 资质审核
   
2. **api/routes/admin/export.js** (导出接口)
   - GET /api/admin/export/qualifications - 资质数据导出

3. **api/routes/admin/dashboard.js** (统计接口)
   - GET /api/admin/dashboard/overview - 概览统计
   - GET /api/admin/dashboard/realtime - 实时数据
   - GET /api/admin/dashboard/stats - 统计数据

### 修改详情

| 修改项 | 修改前 | 修改后 |
|-------|--------|--------|
| 集合名称 | `db.collection('certificates')` | `db.collection('qualifications')` |
| 变量命名 | `certificates`, `certificate` | `qualifications`, `qualification` |
| 错误码 | `CERTIFICATE_*` | `QUALIFICATION_*` |
| 日志字段 | `certificateId`, `certificateUserId` | `qualificationId`, `qualificationUserId` |
| 用户字段 | `certificates` (数组) | `qualifications` (数组) |
