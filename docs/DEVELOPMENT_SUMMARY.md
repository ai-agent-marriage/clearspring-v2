## 🔧 二、遇到的技术问题

### 问题 1：子代理配对失败（Phase 1 初期）

**现象**：
```
gateway closed (1008): pairing required
Gateway target: ws://127.0.0.1:18789
```

**原因**：
- Gateway 绑定在 loopback 模式
- `session.dmScope` 设置为 `per-channel-peer`
- 子代理调用需要配对验证

**解决方案**：
1. 修改 `session.dmScope` 为 `per-peer`
2. 修改飞书 `dmPolicy` 为 `open`
3. 重启 Gateway

**耗时**：30 分钟  
**经验**：多 Agent 协同前必须先验证 Gateway 配置

---

### 问题 2：PC 管理后台质量评分低（Phase 1 Day 3）

**现象**：质量评分 67.5/100，未达 80 分红线

**问题**：
- Element Plus 未集成
- 路由配置缺失
- API 封装缺失
- 9 个业务页面未开发

**解决方案**：

2. ElementPlus 集成-Agent
3. 路由配置-Agent
4. API 封装-Agent
5. 页面开发-Agent
6. 质量验证-Agent

**结果**：质量评分提升至 85/100  
**耗时**：40 分钟  
**经验**：多 Agent 并行加速 4-5 倍

---

### 问题 3：HTTPS 证书申请失败（Phase 3）

**现象**：Let's Encrypt HTTP 验证失败

**原因**：
- 端口 80 被 Nginx 占用
- Certbot 无法创建验证文件

**解决方案**：
1. 改用 DNS 手动验证模式
2. 添加 TXT 记录到 DNS
3. 等待 DNS 传播后继续

**耗时**：20 分钟  
**经验**：生产环境优先使用 DNS 验证

---

### 问题 4：域名解析错误（Phase 3）

**现象**：`sf.dexoconnect.com` 解析到错误 IP（8.221.123.101）

**解决方案**：
1. 更换新域名 `springs.dexoconnect.com`
2. 添加 A 记录到 101.96.192.63
3. 重新申请 HTTPS 证书

**耗时**：15 分钟  
**经验**：新域名无缓存问题，更顺利

---

### 问题 5：管理后台 API 路径错误（Phase 3）

**现象**：登录显示"接口不存在"

**原因**：前端调用 `/api/admin/login`，后端路由是 `/api/admin/auth/login`

**解决方案**：
1. 统一 API 路径规范
2. 更新前端登录页
3. 添加 API 路径文档

**耗时**：10 分钟  
**经验**：前后端路径规范必须提前统一

---

### 问题 6：Nginx 权限问题（Phase 3）

**现象**：管理后台 500 错误

**原因**：
- `/root` 目录权限 700
- Nginx worker 进程（www-data）无法访问

**解决方案**：
1. 将静态文件移到 `/var/www/clearspring-admin/`
2. 设置正确权限（755）
3. 所有者改为 www-data

**耗时**：20 分钟  
**经验**：Web 资源不要放在/root 下
