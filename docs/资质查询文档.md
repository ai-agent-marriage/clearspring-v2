## 🔗 API 接口

### 1. 获取资质列表

**接口地址**: `GET /api/admin/qualifications`

**权限要求**: 管理员（admin 角色）

**请求参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|-------|------|------|--------|------|
| page | Number | 否 | 1 | 页码，从 1 开始 |
| pageSize | Number | 否 | 20 | 每页数量 |
| status | String | 否 | - | 资质状态筛选 |
| type | String | 否 | - | 资质类型筛选 |
| userId | String | 否 | - | 用户 ID 筛选 |

**状态枚举值**:
- `pending` - 待审核
- `approved` - 已通过
- `rejected` - 已驳回

**请求示例**:

```bash
# 获取资质列表（默认分页）
curl -X GET "https://springs.dexoconnect.com/api/admin/qualifications" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -k

# 分页查询
curl -X GET "https://springs.dexoconnect.com/api/admin/qualifications?page=2&pageSize=50" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -k

# 按状态筛选
curl -X GET "https://springs.dexoconnect.com/api/admin/qualifications?status=pending" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -k

# 组合筛选
curl -X GET "https://springs.dexoconnect.com/api/admin/qualifications?status=pending&type=driver&userId=xxx" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -k
```

**响应示例**:

### 2. 审核资质

**接口地址**: `PUT /api/admin/qualifications/:id`

**权限要求**: 管理员（admin 角色）

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| id | String | 是 | 资质 ID（qualificationId） |

**请求体参数**:

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| status | String | 是 | 审核状态：approved / rejected |
| rejectReason | String | 条件必填 | 驳回原因（status=rejected 时必填） |
| auditRemark | String | 否 | 审核备注 |

**请求示例**:

```bash
# 审核通过
curl -X PUT "https://springs.dexoconnect.com/api/admin/qualifications/507f1f77bcf86cd799439011" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "approved"}' \
  -k

# 审核驳回
curl -X PUT "https://springs.dexoconnect.com/api/admin/qualifications/507f1f77bcf86cd799439011" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "rejected", "rejectReason": "图片不清晰，请重新上传", "auditRemark": "请联系客服"}' \
  -k
```

**响应示例**（审核通过）:

```json
{
  "code": "SUCCESS",
  "data": {
    "qualificationId": "507f1f77bcf86cd799439011",
    "status": "approved",
    "auditTime": "2026-03-30T11:20:00.000Z"
  },
  "message": "审核通过"
}
```

**响应示例**（审核驳回）:

```json
{
  "code": "SUCCESS",
  "data": {
    "qualificationId": "507f1f77bcf86cd799439011",
    "status": "rejected",
    "auditTime": "2026-03-30T11:20:00.000Z"
  },
  "message": "已驳回"
}
```

## 🔍 查询逻辑详解

### 1. 动态查询条件构建

```javascript
const query = {};

if (status) {
  query.status = status; // pending, approved, rejected
}

if (type) {
  query.type = type;
}

if (userId) {
  query.userId = new ObjectId(userId);
}
```

**说明**:
- 所有参数都是可选的
- 不传参数时查询全部资质
- 多个条件同时存在时使用 AND 逻辑

### 2. 分页逻辑

```javascript
const skip = (parseInt(page) - 1) * parseInt(pageSize);
const limit = parseInt(pageSize);
```

**示例**:
- page=1, pageSize=20 → skip=0, limit=20
- page=2, pageSize=20 → skip=20, limit=20
- page=3, pageSize=50 → skip=100, limit=50

### 3. 排序规则

```javascript
.sort({ createdAt: -1 })
```

**说明**: 按创建时间倒序排列，最新的资质排在最前面

### 4. 用户信息关联查询

```javascript
// 提取所有 userId
const userIds = qualifications
  .filter(q => q.userId)
  .map(q => q.userId);

// 批量查询用户信息
const users = await db.collection('users')
  .find({ _id: { $in: userIds } })
  .project({ 
    _id: 1, 
    nickName: 1, 
    phone: 1, 
    avatarUrl: 1 
  })
  .toArray();

// 构建用户映射
const userMap = {};
users.forEach(user => {
  userMap[user._id.toString()] = user;
});
```

**优化说明**:
- 使用批量查询而非逐条查询，提高性能
- 只查询需要的字段，减少数据传输
- 使用 Map 结构快速匹配用户信息

## 📊 数据库集合

### qualifications 集合结构

```javascript
{
  _id: ObjectId,                    // 资质 ID
  userId: ObjectId,                 // 用户 ID（关联 users 集合）
  type: String,                     // 资质类型
  certificateNo: String,            // 证书编号
  certificateName: String,          // 证书名称
  issueDate: Date,                  // 颁发日期
  expiryDate: Date,                 // 到期日期
  images: [String],                 // 证书图片 URL 数组
  status: String,                   // 状态：pending | approved | rejected
  rejectReason: String,             // 驳回原因
  auditUserId: ObjectId,            // 审核人 ID（管理员）
  auditTime: Date,                  // 审核时间
  auditRemark: String,              // 审核备注
  createdAt: Date,                  // 创建时间
  updatedAt: Date                   // 更新时间
}
```

### 推荐索引

```javascript
// 复合索引：常用查询组合
db.qualifications.createIndex({ status: 1, createdAt: -1 });

// 单字段索引
db.qualifications.createIndex({ userId: 1 });
db.qualifications.createIndex({ type: 1 });
db.qualifications.createIndex({ createdAt: -1 });

// 覆盖索引：优化特定查询
db.qualifications.createIndex({ status: 1, userId: 1, createdAt: -1 });
```

## ⚠️ 错误处理

### 常见错误码

| 错误码 | HTTP 状态码 | 说明 |
|-------|-----------|------|
| INVALID_STATUS | 400 | 无效的状态值 |
| MISSING_REJECT_REASON | 400 | 驳回时未填写原因 |
| QUALIFICATION_NOT_FOUND | 404 | 资质记录不存在 |
| QUALIFICATION_ALREADY_AUDITED | 400 | 资质已审核过 |
| FORBIDDEN | 403 | 权限不足 |

### 错误响应示例

```json
{
  "code": "QUALIFICATION_NOT_FOUND",
  "message": "资质记录不存在"
}
```

```json
{
  "code": "INVALID_STATUS",
  "message": "无效的状态，只能是 approved 或 rejected"
}
```

## 🔐 权限说明

### 管理员权限验证流程

1. 用户登录获取 Token
2. 请求携带 Token：`Authorization: Bearer <token>`
3. `authMiddleware` 验证 Token 有效性
4. `adminMiddleware` 验证用户角色是否为 `admin`
5. 验证通过后执行查询逻辑

### 权限不足响应

```json
{
  "code": "FORBIDDEN",
  "message": "权限不足，需要管理员权限"
}
```

## 🚀 性能优化建议

1. **分页大小**: 建议 pageSize 不超过 100，避免单次查询过多数据
2. **索引优化**: 根据实际查询频率添加合适的索引
3. **批量查询**: 用户信息使用批量查询而非循环查询
4. **字段投影**: 只查询需要的字段，减少网络传输
5. **缓存策略**: 对于不常变化的数据可考虑缓存
