# 资质审核接口修复报告

## 📋 修复概述

**修复日期**: 2026-03-30  
**问题类型**: 数据库集合名称错误  
**影响范围**: 资质审核相关 API（7 个测试用例全部失败）  
**修复状态**: ✅ 已完成

---

## 🔍 问题分析

### 问题现象
- 资质审核 API 返回 `INTERNAL_ERROR`
- 7 个测试用例全部失败
- 数据库查询无法获取数据

### 根本原因
代码中使用的数据库集合名称不一致：
- **错误使用**: `certificates` 集合
- **应该使用**: `qualifications` 集合

这导致 API 查询时访问了错误的集合，从而返回空数据或错误。

## 🔧 修复内容

### 1. 核心接口修复

#### 文件：`api/routes/admin/qualifications.js`

**GET /api/admin/qualifications** - 资质列表查询

修复前：
```javascript
const certificates = await db.collection('certificates')
  .find(query)
  .sort({ createdAt: -1 })
  .skip(skip)
  .limit(limit)
  .toArray();

const total = await db.collection('certificates').countDocuments(query);
```

修复后：
```javascript
const qualifications = await db.collection('qualifications')
  .find(query)
  .sort({ createdAt: -1 })
  .skip(skip)
  .limit(limit)
  .toArray();

const total = await db.collection('qualifications').countDocuments(query);
```

**PUT /api/admin/qualifications/:id** - 资质审核

修复前：

## 📊 数据库集合结构

### qualifications 集合

```javascript
{
  _id: ObjectId,
  userId: ObjectId,           // 用户 ID
  type: String,               // 资质类型
  certificateNo: String,      // 证书编号
  certificateName: String,    // 证书名称
  issueDate: Date,            // 颁发日期
  expiryDate: Date,           // 到期日期
  images: [String],           // 证书图片 URL 数组
  status: String,             // pending | approved | rejected
  rejectReason: String,       // 驳回原因
  auditUserId: ObjectId,      // 审核人 ID
  auditTime: Date,            // 审核时间
  auditRemark: String,        // 审核备注
  createdAt: Date,            // 创建时间
  updatedAt: Date             // 更新时间
}
```
