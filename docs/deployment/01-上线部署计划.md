## 🚀 上线步骤

### 步骤 1：代码部署

```bash
# 1. 拉取最新代码
cd /opt/clearspring-api
git pull origin main

# 2. 安装依赖
npm install --production

# 3. 构建项目
npm run build

# 4. 重启 PM2 进程
pm2 restart clearspring-api
pm2 save
```

### 步骤 2：数据库初始化

```bash
# 1. 备份现有数据库
mysqldump -u [用户名] -p clearspring > backup_20260401.sql

# 2. 执行数据库迁移
cd /opt/clearspring-api
npm run migrate

# 3. 验证数据完整性
npm run db:verify
```

### 步骤 3：配置检查

- [ ] 检查 `.env` 文件配置是否正确
- [ ] 验证数据库连接字符串
- [ ] 确认第三方 API 密钥已配置
- [ ] 检查日志级别设置
- [ ] 验证 SMTP 邮件服务配置

## 🔄 回滚方案

### 回滚触发条件

1. 核心功能无法正常使用
2. 数据库出现严重错误
3. 系统性能严重下降（响应时间 > 5 秒）
4. 安全漏洞被发现

### 回滚步骤

```bash
# 1. 停止当前服务
pm2 stop clearspring-api

# 2. 回滚代码到上一版本
cd /opt/clearspring-api
git revert HEAD
# 或
git checkout [上一个稳定版本 tag]

# 3. 恢复数据库备份
mysql -u [用户名] -p clearspring < backup_20260401.sql

# 4. 重启服务
pm2 start clearspring-api
pm2 save

# 5. 验证回滚成功
curl -X GET https://api.clearspring.com/health
```

### 回滚验证

- [ ] API 健康检查通过
- [ ] 数据库连接正常
- [ ] 核心功能可正常使用
- [ ] 用户可正常访问

### 常见问题处理

#### 1. 服务无法启动

```bash
# 查看 PM2 日志
pm2 logs clearspring-api

# 查看系统资源
top
df -h
free -m

# 重启服务
pm2 restart clearspring-api
```

#### 2. 数据库连接失败

```bash
# 检查数据库服务状态
systemctl status mysql

# 测试数据库连接
mysql -h [数据库地址] -u [用户名] -p

# 检查防火墙
iptables -L -n | grep 3306
```

#### 3. API 响应缓慢

```bash
# 查看慢查询日志
tail -f /var/log/mysql/slow.log

# 检查系统负载
uptime
vmstat 1 5

# 查看应用性能监控
# [APM 系统链接]
```
