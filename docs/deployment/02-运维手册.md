## 📊 PM2 进程管理

### 进程信息

| 进程名称 | 说明 | 实例数 | 内存限制 |
|----------|------|--------|----------|
| clearspring-api | 主 API 服务 | 4 | 2GB |

### 常用命令

```bash
# 查看进程状态
pm2 status
pm2 status clearspring-api

# 查看进程详情
pm2 describe clearspring-api

# 重启进程
pm2 restart clearspring-api
pm2 restart clearspring-api --update-env

# 停止进程
pm2 stop clearspring-api

# 启动进程
pm2 start clearspring-api

# 删除进程
pm2 delete clearspring-api

# 查看日志
pm2 logs clearspring-api
pm2 logs clearspring-api --lines 100

# 实时监控
pm2 monit

# 保存进程列表（开机自启）
pm2 save

# 加载保存的进程列表
pm2 resurrect
```

### PM2 配置文件

位置：`/opt/clearspring-api/ecosystem.config.js`

```javascript
module.exports = {
  apps: [{
    name: 'clearspring-api',
    script: './dist/main.js',
    instances: 4,
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    max_memory_restart: '2G',
    error_file: '/opt/clearspring-logs/api/error.log',
    out_file: '/opt/clearspring-logs/api/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss',
    merge_logs: true,
    autorestart: true,
    watch: false
  }]
};
```

### 开机自启配置

```bash
# 生成开机自启脚本
pm2 startup systemd

# 执行输出的命令，例如：
# sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root

# 保存当前进程列表
pm2 save
```

## 📝 日志查看方法

### PM2 日志

```bash
# 实时查看所有日志
pm2 logs

# 查看指定进程日志
pm2 logs clearspring-api

# 查看最近 100 行
pm2 logs clearspring-api --lines 100

# 查看错误日志
pm2 logs clearspring-api --err

# 带时间戳查看
pm2 logs clearspring-api --timestamp
```

### 应用日志

日志位置：`/opt/clearspring-logs/api/`

```bash
# 实时查看日志
tail -f /opt/clearspring-logs/api/app.log

# 查看最近 1000 行
tail -n 1000 /opt/clearspring-logs/api/app.log

# 搜索关键字
grep "ERROR" /opt/clearspring-logs/api/app.log
grep "2026-04-01" /opt/clearspring-logs/api/app.log

# 按日期查看
ls -lh /opt/clearspring-logs/api/
```

### 系统日志

```bash
# 查看系统日志
journalctl -xe

# 查看最近启动日志
journalctl -b

# 查看 Nginx 日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log

# 查看 MySQL 日志
tail -f /var/log/mysql/error.log
```

### 日志轮转

日志轮转配置：`/etc/logrotate.d/clearspring`

```
/opt/clearspring-logs/api/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0640 root root
    postrotate
        pm2 reload clearspring-api > /dev/null
    endscript
}
```

### 告警配置

#### 1. 服务器资源告警

监控指标：
- CPU 使用率 > 80% 持续 5 分钟
- 内存使用率 > 85% 持续 5 分钟
- 磁盘使用率 > 90%
- 磁盘 inode 使用率 > 90%

#### 2. 服务可用性告警

```bash
# 健康检查脚本
#!/bin/bash
RESPONSE=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/health)
if [ "$RESPONSE" != "200" ]; then
    echo "API health check failed: $RESPONSE"
    # 发送告警通知
    # [集成飞书/短信/电话告警]
fi
```

#### 3. 数据库监控

```sql
-- 查看连接数
SHOW STATUS LIKE 'Threads_connected';

-- 查看慢查询
SHOW VARIABLES LIKE 'slow_query_log';
SHOW VARIABLES LIKE 'long_query_time';
```

### 告警通知渠道

| 级别 | 渠道 | 响应时间要求 |
|------|------|--------------|
| P0 - 严重 | 电话 + 短信 + 飞书 | 5 分钟内 |
| P1 - 高 | 短信 + 飞书 | 15 分钟内 |

## 🔒 安全运维

### 防火墙配置

```bash
# 查看防火墙状态
ufw status

# 允许 SSH
ufw allow 22/tcp

# 允许 HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 启用防火墙
ufw enable
```

### SSH 安全加固

```bash
# 编辑 SSH 配置
vim /etc/ssh/sshd_config

# 建议配置：
# PermitRootLogin prohibit-password
# PasswordAuthentication no
# PubkeyAuthentication yes
# MaxAuthTries 3
```

### 定期安全扫描

```bash
# 安装安全扫描工具
apt install lynis

# 执行安全审计
lynis audit system
```
