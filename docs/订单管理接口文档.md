# 订单管理接口文档

## 概述

订单管理接口提供管理员对平台订单的查询、状态更新和删除功能。

**基础 URL**：`https://springs.dexoconnect.com/api/admin`

**认证方式**：Bearer Token（需要管理员权限）

---

## 接口列表

### 1. 获取订单列表

**接口**：`GET /orders`

**权限**：管理员（admin）

**查询参数**：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | number | 否 | 1 | 页码 |
| pageSize | number | 否 | 20 | 每页数量 |
| status | string | 否 | - | 订单状态筛选 |
| paymentStatus | string | 否 | - | 支付状态筛选 |
| serviceType | string | 否 | - | 服务类型筛选 |
| startDate | string | 否 | - | 开始日期（ISO 8601） |
| endDate | string | 否 | - | 结束日期（ISO 8601） |
| keyword | string | 否 | - | 关键词搜索 |

**订单状态**：
- `pending` - 待支付
- `paid` - 已支付
- `grabbed` - 已接单
- `completed` - 已完成
- `cancelled` - 已取消

**支付状态**：
- `unpaid` - 未支付
- `paid` - 已支付
- `refunded` - 已退款

**请求示例**：

```bash
curl -X GET "https://springs.dexoconnect.com/api/admin/orders?page=1&pageSize=10&status=paid" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

### 2. 更新订单状态

**接口**：`PUT /order/:id/status`

**权限**：管理员（admin）

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 订单 ID |

**请求体**：

```json
{
  "status": "completed",
  "remark": "用户确认完成"
}
```

**请求示例**：

```bash
curl -X PUT "https://springs.dexoconnect.com/api/admin/order/507f1f77bcf86cd799439011/status" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status":"completed","remark":"用户确认完成"}'
```

**响应示例**：

```json
{
  "code": "SUCCESS",
  "data": {
    "orderId": "507f1f77bcf86cd799439011",
    "status": "completed",
    "updatedAt": "2026-03-30T12:00:00.000Z"
  },
  "message": "订单状态已更新"
}
```

**错误响应**：

```json
{
  "code": "ORDER_NOT_FOUND",
  "data": null,
  "message": "订单不存在"
}
```

```json
{
  "code": "INVALID_STATUS",
  "data": null,
  "message": "无效的订单状态"
}
```

### 3. 删除订单

**接口**：`DELETE /order/:id`

**权限**：管理员（admin）

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 订单 ID |

**限制**：只能删除已取消（cancelled）状态的订单

**请求示例**：

```bash
curl -X DELETE "https://springs.dexoconnect.com/api/admin/order/507f1f77bcf86cd799439011" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

**响应示例**：

```json
{
  "code": "SUCCESS",
  "data": {
    "orderId": "507f1f77bcf86cd799439011",
    "orderNo": "ORD202603300001",
    "deletedAt": "2026-03-30T12:00:00.000Z"
  },
  "message": "订单已删除"
}
```

**错误响应**：

```json
{
  "code": "ORDER_CANNOT_DELETE",
  "data": null,
  "message": "只能删除已取消的订单"
}
```

## 错误码说明

| 错误码 | HTTP 状态 | 说明 |
|--------|----------|------|
| SUCCESS | 200 | 成功 |
| UNAUTHORIZED | 401 | 未提供认证令牌 |
| INVALID_TOKEN | 401 | 无效的认证令牌 |
| TOKEN_EXPIRED | 401 | 认证令牌已过期 |
| FORBIDDEN | 403 | 权限不足 |
| ORDER_NOT_FOUND | 404 | 订单不存在 |
| INVALID_STATUS | 400 | 无效的订单状态 |
| ORDER_CANNOT_DELETE | 400 | 订单不可删除 |
| INTERNAL_ERROR | 500 | 服务器内部错误 |

## 数据库查询逻辑

### 查询构建

```javascript
// 基础查询
const query = {};

// 状态筛选
if (status) {
  query.status = status;
}

// 支付状态筛选
if (paymentStatus) {
  query.paymentStatus = paymentStatus;
}

// 服务类型筛选
if (serviceType) {
  query.serviceType = serviceType;
}

// 时间范围筛选
if (startDate || endDate) {
  query.serviceDate = {};
  if (startDate) {
    query.serviceDate.$gte = new Date(startDate);
  }
  if (endDate) {
    query.serviceDate.$lte = new Date(endDate);
  }
}

// 关键词搜索（订单号、服务名称、地点）
if (keyword) {
  const keywordRegex = new RegExp(keyword, 'i');
  query.$or = [
    { orderNo: keywordRegex },
    { serviceName: keywordRegex },
    { location: keywordRegex }
  ];
}
```

### 分页查询

```javascript
const skip = (parseInt(page) - 1) * parseInt(pageSize);
const limit = parseInt(pageSize);

const orders = await db.collection('orders')
  .find(query)
  .sort({ createdAt: -1 })
  .skip(skip)
  .limit(limit)
  .toArray();

const total = await db.collection('orders').countDocuments(query);
```

### 用户信息关联

```javascript
// 批量获取用户信息
const userIds = orders.filter(o => o.userId).map(o => o.userId);
const users = await db.collection('users')
  .find({ _id: { $in: userIds } })
  .project({ _id: 1, nickName: 1, phone: 1 })
  .toArray();

// 批量获取执行者信息
const executorIds = orders.filter(o => o.executorId).map(o => o.executorId);
const executors = await db.collection('users')
  .find({ _id: { $in: executorIds } })
  .project({ _id: 1, nickName: 1, phone: 1 })
  .toArray();
```

## 最佳实践

### 1. 分页查询

建议使用合理的分页大小（10-50 条），避免一次性加载大量数据。

```bash
# 推荐：每页 20 条
GET /api/admin/orders?page=1&pageSize=20

# 不推荐：每页 1000 条
GET /api/admin/orders?page=1&pageSize=1000
```

### 2. 条件筛选

尽量使用具体的筛选条件，减少数据库查询压力。

```bash
# 推荐：按状态筛选
GET /api/admin/orders?status=paid&pageSize=20

# 推荐：按时间范围筛选
GET /api/admin/orders?startDate=2026-03-01&endDate=2026-03-31
```

### 3. 关键词搜索

关键词搜索支持模糊匹配，但性能较低，建议与其他条件组合使用。

```bash
# 推荐：组合使用
GET /api/admin/orders?keyword=ORD&status=paid

# 不推荐：单独使用（可能匹配大量数据）
GET /api/admin/orders?keyword=测试
```
