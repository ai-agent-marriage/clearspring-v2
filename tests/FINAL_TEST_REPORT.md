## 🔍 4. 失败用例分析

### 4.1 Dashboard 统计模块 (8 个失败)

**失败用例**:
- API-DASH-001 ~ API-DASH-011 (全部失败)

**错误信息**: `Expected: "SUCCESS", Received: "INTERNAL_ERROR"`

**原因分析**:
- Dashboard 路由使用 `req.app.get('db')` 获取数据库连接
- 聚合查询可能因集合不存在或数据结构不匹配而失败
- 建议检查：数据库集合是否存在、字段名是否匹配

### 4.2 订单管理模块 (12 个失败)

**失败用例**:
- API-ORD-001 ~ API-ORD-016 (大部分失败)

**错误信息**: 
- `Expected: "SUCCESS", Received: "INTERNAL_ERROR"`
- `TypeError: Cannot read properties of null (reading 'orders')`

**原因分析**:
- 订单列表接口返回 INTERNAL_ERROR 导致后续测试无法获取订单 ID
- 级联失败：依赖前置接口的测试全部失败

### 4.3 资质审核模块 (7 个失败)

**失败用例**:
- API-QUAL-001 ~ API-QUAL-011 (大部分失败)

**错误信息**: 同上，INTERNAL_ERROR

**原因分析**: 数据库查询失败

### 4.4 执行者管理模块 (5 个失败)

**通过用例**: API-EXEC-008 (无效状态验证)

### 4.5 分账配置模块 (7 个失败)

**失败用例**: API-PROF-001 ~ API-PROF-008 (全部失败)

### 4.6 数据导出模块 (5 个失败)

**失败用例**: API-EXPT-001 ~ API-EXPT-007

**错误信息**: 
- `Expected substring: "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"`
- `Received string: "application/json; charset=utf-8"`

**原因分析**: 
- 导出接口返回 JSON 而非文件流
- 文件生成逻辑未实现或响应头设置错误

### 4.7 操作日志模块 (4 个失败)

- 路由文件存在但未在 index.js 中引用
