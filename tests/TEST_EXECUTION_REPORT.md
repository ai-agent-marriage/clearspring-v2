## 🔍 问题分析

### 1. 认证错误处理不一致
**问题**: 未认证时，部分接口返回 500 而非预期的 200 + 错误码

**影响**: 10 个测试失败

**建议**: 
- 统一错误处理中间件
- 确保所有认证错误返回 200 + 错误码或统一返回 401

### 2. 资源不存在响应
**问题**: 资源不存在时返回 404 HTTP 状态码，测试期望 200 + 错误码

**影响**: 6 个测试失败

**建议**:
- 统一 API 响应风格
- 更新测试以匹配实际行为

### 3. 登录响应格式
**问题**: 登录成功返回的数据结构与测试预期不同
```javascript
// 实际返回
{
  data: {
    token: "...",
    admin: { adminId, username, role, ... }
  }
}

// 测试期望
{
  data: {
    token: "...",
    username: "...",
    role: "..."
  }
}
```

**影响**: 1 个测试失败

**建议**: 更新测试断言

### 4. 数据导出接口路径
**问题**: 所有导出接口返回 404

**影响**: 5 个测试失败

**建议**: 确认实际接口路径（可能是 `/api/admin/export/orders` 而非其他）
